# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

An automated library-cataloguing agent: it ingests book/magazine files (EPUB, PDF, scanned image sets — pre- or post-ISBN), extracts and enriches their metadata, classifies them by **CDU** (Universal Decimal Classification), and persists schema-validated documents to **MongoDB Atlas**. It is meant to run in Docker on a NAS with an `Inbox` folder mapped under the app root. Code, comments, and identifiers are in **Spanish** — match that convention.

> **Durable project context lives in [`docs/contexto-proyecto.md`](docs/contexto-proyecto.md)** — the accumulated *why* behind decisions, the non-obvious **NAS/Atom deployment constraints** (no sharp/SIMD, no Alpine, Node 18/cheerio 1.0.0, Compose v1, chokidar-over-bind-mount, LF endings, the `down -v` gotcha, the deploy script + `nas-estable` rollback tag), the governing principles (conservative merge, ISBN pivot), and the subsystem history (covers, scanned-PDF OCR, transactional copy, the Conformador). Read it before non-trivial work or any deploy.

## Commands

- **Run the app** (REST API on port 3000 **+** Inbox watcher): `npm start`. The watcher **starts PAUSED by default** (observes the Inbox but does not catalogue until you activate it from the Control Panel's Vigilante switch); set `VIGILANTE_AUTOSTART=1` to start it already active. Set `DESACTIVAR_VIGILANTE=1` to not start the watcher at all (API only). ⚠️ Run **one instance at a time** — two watchers race on the Inbox and cause ENOENT mid-copy.
- **Test suite** = the battery of 12 real-world cases in `Test Battery/`: `node "Test Battery/run-battery.js"`. **Run a single case (or subset):** pass comma-separated ids, e.g. `node "Test Battery/run-battery.js" 12` or `1,3,12`. Cases 1–3 EPUB, 4–7 PDF, 8–11 images, 12 PDF magazine. The runner clears nothing — it inserts/updates Mongo and writes a JSON per case folder under `_resultado/`.
- **Isolated checks** (also in `Test Battery/`): `test-credenciales.js` (verify Gemini + Google Books keys), `test-conservador.js` (conservative merge), `test-vision-api.js` (vision + APIs), `test-endpoint.js` / `test-inbox.js` (live ingest methods — require `npm start` running).
- **Backfill `registro.json` + `registro.marc.xml`** for all catalogued docs: `node scripts/regenerar-registros.js [--ejecutar]` (dry-run by default). Note: `Test Battery/` is **gitignored**, so it does NOT deploy to the NAS — maintenance/recovery scripts that must run there live in the tracked `scripts/` folder (auditoria-integridad, restaurar-originales, re-enriquecer-degradados, reingestar-multiarchivo, regenerar-registros, setup-mongo, **resolver-duplicados** — auto-resolves the Cuarentena/duplicados backlog: same-hash→delete, diff-format→both, else keep bigger/most-recent + DB sync; dry-run by default, **retirar-bne-cdus**, **etl-fichero**, **integridad** — the consolidated integrity task, **reindexar-busqueda** — rebuild the local FTS search index, **reclasificar-editoriales** — batch-fix the `editorial` field of a set of books (`--editorial "ePubLibre"`/`--id`/`--sin-editorial`/`--todos`) via cascade lookup Fichero→OpenLibrary→Google→AI(opt-in `--ia`), report by transition, dry-run by default, **reparar-portadas** — finds docs with NO cover or BROKEN carousel images (`imagenes[]` files missing on disk, e.g. after an interrupted ingest): prunes the dead image refs and re-resolves the cover by **reusing the first valid remaining image** or **extracting the first page** of the original file (PDF/EPUB/DjVu/comic via `resolverPortada`); dry-run by default, `--ejecutar`/`--id <id>`).
- **Integrity (diagnose / diagnose+fix)**: `node scripts/integridad.js [--reparar]` — one pass over DB↔disk consolidating every check+repair (`src/integridad.js` · `verificarIntegridad({reparar})`, also fired from the panel's **Integridad** section). Incluye **docsConAudiosRotos** (audiolibros cuyo `audios[]` apunta a ficheros que no están: PUNTO CIEGO histórico, porque la comprobación de «docs sin fichero» EXCLUYE a los audiolibros). Repairs are recycle-only (safe): prune empty/dead branches, make `ruta_base` 1:1 with its folder, recycle orphan folders, dedupe by hash (keep best), resolve Cuarentena/duplicados; docs-without-folder / docs-without-file are **diagnose-only** (manual). Schedule daily/weekly with DSM's Task Scheduler running `docker exec gestor-biblioteca node scripts/integridad.js --reparar`. (The older `auditoria-integridad`/`resolver-duplicados` scripts remain for detailed per-item CLI output.)
- **No build/lint/test framework** — plain Node ESM (`"type": "module"`). Syntax-check a file with `node --check <file>`. There is no transpile step.

## Architecture — the ingest pipeline

Everything flows through one shared service so both entry points behave identically:

```
Entry points:                         Shared pipeline (src/servicio-ingesta.js · ingestarRecurso):
  app.js   POST /api/ingestar  ─┐       orquestador.js  → extract by type
  vigilante.js  Inbox watcher  ─┴──►     motor-enriquecimiento.js → enrich (conservative)
                                         motor-catalogo.js → resolve refs + dedup + upsert
                                         copy files into CDU tree + write registro.json/.marc.xml
```

- **`orquestador.js` (`procesarRecurso`)** routes by file type: EPUB → `utils/lector-epub.js`; PDF → `utils/lector-pdf.js` (text layer + ISBN/ISSN regex; flags scanned PDFs); image group → `agente.js` (Gemini vision). Loose JPEGs are grouped into **one** book by `utils/agrupador.js`. Unknown-but-known formats (mobi/cbr/djvu) are catalogued by filename as a placeholder; a root‑level **`.zip`/`.rar`/`.7z` dropped in the Inbox is EXPANDED** by the watcher into a folder‑drop (`vigilante.js·expandirComprimidos`) so it gets the **same folder logic** (images→one scanned book, mixed files→collection, `Vol N`→obra), and the original zip is recycled to the Papelera (never deleted). Stable‑file check before extracting; corrupt zip → Cuarentena/ilegibles; a single top‑level dir is flattened. **libro vs revista** (`orquestador.js`, robust): `esLibro = isbn || cip`; a doc is a `revista` only if `!esLibro && (esFechada || pareceRevista(titulo) || issn)`. I.e. an **ISBN or a CIP/Dewey block ⇒ book** even with a *series* ISSN (Springer monographs print a «Lecture Notes…» ISSN + a CIP record; that doesn't make them periodicals); the ISSN **alone** never makes a magazine. The ISBN is recovered from the **content, the whole filename, *and* an ISBN embedded in the filename** (`lector-pdf.js` runs `extraerISBNs(nombre)` — catches `9783…(1).pdf`, `<isbn> Title.pdf`). Producer-watermark titles (`Creator_ Advanced PDF Repair…`, `PScript5.dll`, a URL) are rejected by `parsear-nombre.js·esTituloArtefacto` → fall back to the filename/authority.
- **Tiered enrichment (cost-minimizing): file → local Fichero → free APIs → AI.** `utils/proveedor-metadatos.js` orchestrates **Tier 2.0 = `utils/buscador-local.js`** (the offline OL+BNE dump in `Fichero/fichero.db`, queried by ISBN first — see §"Fichero" in `docs/contexto-proyecto.md`), then the online Tier 2 as *freshness fallback* (`buscador-bibliografico.js`=OpenLibrary — **skipped on a local hit**; `buscador-google-books.js`; `buscador-dnb.js` + `buscador-bnf.js` SRU for Dewey/CDU), then Tier 3 (Gemini vision for ISBN hints, CDU classification). (The online BNE — old `buscador-bne.js` SPARQL + the Mongo `bne_cdus` cache — was **retired**: the Fichero holds the full BNE dump, so it was pure redundancy on Atlas's free tier.) The **ISBN is the pivot** — get it cheaply, then a lookup resolves the rest. (British National Bibliography is a planned sibling fallback once its Share Family endpoint ships.) The Fichero `.db` is built once by `scripts/etl-fichero.js` (resumable; schema is the single source of truth in `scripts/etl-map.js`).
- **`motor-enriquecimiento.js` is the single conservative-merge authority.** Data extracted from the file is the source of truth and is **never** overwritten by internet/AI — external sources only fill gaps (`primerValido`). Exceptions encoded here: ePubLibre/Lectulandia etc. are NOT publishers (`EDITORIALES_NO_VALIDAS`), so a real API publisher overrides them; and the ISBN inside the provider lets APIs outrank vision. Reader-internal/null fields are stripped before persisting (avoids base64 bloat and `$jsonSchema` rejections).
- **`motor-catalogo.js`** resolves `autores`/`editorial` strings → ObjectId refs (check-then-create in their collections), dedups by **hash → isbn(+format) → (revistas: cabecera+clave) → titulo**, and on a hit performs an intelligent merge (gap-fill + `pendiente`→`completado` upgrade) — this is the "better info found later" update path. **Magazines & series = cabecera/serie-as-colección (ISSN pivot):** a magazine masthead (e.g. *Historia de Iberia Vieja*, ISSN 1699-7913) is modelled as a **`colección` with `tipo:'revista'` + `issn`** (authority, analogous to `obras.isbn_obra`); **each issue is a member doc** (`doc.coleccion` + `clave_numero`) whose identity is **(colección, `clave_numero`)** — `clave_numero` = `AAAA-MM` → `n<nº>` → `AAAA` (`utils/revistas.js` · `claveNumero`/`tituloCabecera`/`clasificarISSN`; `utils/colecciones.js` · `resolverCabecera`/`registrarNumeroEnColeccion`). The cabecera keeps a **chronological** `numeros[]` inventory (not the 1..N array books use); an issue with ISSN but no detectable date/number is attached as a **`numeros_sin_fecha`** member (never merged, never lost). **A book series sharing a *series* ISSN** (e.g. Springer's *Graduate Texts in Physics*, 1868-4513) uses the **same colección model with `tipo:'libro'`**: each book keeps its **own ISBN** and the ISSN lives on the colección (never on the book), so a series ISSN never turns a book into a magazine (`motor-catalogo.js` step 2e). `obras` is now **multivolume books only**; migrate legacy obras-cabecera data with `scripts/migrar-revistas-a-colecciones.js` (dry-run by default). The old `issn`-only dedup fallback (which fused *all* issues of a magazine into one document) was **removed**. **ANTI-LOSS POLICY (never replace/merge-away a document; only an exact-hash copy is ever deleted):** the **only** deletion is a **same-hash** re-drop of an already-catalogued file (obviously identical → deleted). Everything else becomes **its own document**: if a dedup key (`isbn(+format)`, `(coleccion,clave)`, `(obra,volumen)`) lands on a doc that already has a *different* file, it is **NOT** merged — it inserts as a distinct doc (`motor-catalogo.js` guards the merge on `nombre_archivo`). A same-identifier/different-content arrival is **kept** in `Cuarentena/duplicados` for manual dedup (no auto delete/replace — the old size-based `ganaEntrante`/`reemplazarFicheroDeDoc` auto-resolution was **removed**). **1 document ↔ 1 folder:** the `_id`-suffix folder disambiguation (`rutas.js` `discriminador`) applies to **revistas too**, so two issues that would share `revistas/<cabecera>/<año>` (e.g. same year, no month) each get their own folder and never clobber each other's sidecars. Dedup-by-format is now *informational* (you may still get one doc per pdf/epub/mobi), not a merge/replace trigger.

**ISSN capture for magazines (when not in the text):** (1) **cover barcode** — `utils/lector-barras.js` crops the cover/back barcode zones at high res with poppler (Atom-safe, C). It first tries to read the EAN **LOCALLY, no AI** with **`utils/lector-barras-local.js` (zxing-wasm)** — a free, offline pre-step that saves a vision call (and 429s); only if that fails does it fall back to Gemini-on-focused-crop. zxing-wasm is WASM (loaded with a local `wasmBinary` override in Node) and **degrades gracefully** — if its `.wasm` won't instantiate (e.g. needs SIMD the Atom lacks) the local read returns null and vision takes over, never breaking ingest. Then `977`→ISSN (forces `tipo:'revista'`), `978/979`→ISBN, EAN-2 add-on (1-12)→month (`utils/codigo-barras.js` · `decodificarCodigoBarras`). (2) **title→ISSN** — `utils/buscador-issn-titulo.js` (Wikidata, keyless; only serials carry P236) as a last resort in `motor-enriquecimiento` for barcode-less issues. **Why ingest is cheap and the Conformador is thorough:** ingest is throughput/cost-optimized (the watcher must keep up; APIs/AI are rate-limited/paid), while the Conformador perfects records at idle — *and* it can use **cross-document context that only exists later** (a series' title learned from a sibling volume, the `equivalencias_cdu` cache, a resolved ISSN cabecera). The conservative merge guarantees "good enough now, perfected later" never degrades a record. Opt-in **`CONFORMAR_AL_INGERIR=1`** (or `POST /api/ingestar` with `conformar=1`) runs the Conformador on the just-catalogued doc, **fire-and-forget** (`mantenimiento/conformador.js` · `conformarAlIngerir`) — great for single manual drops; it gets ~80% of the gain (the cross-document part still waits for the sibling).

## CDU classification & the `equivalencias_cdu` cache

`src/clasificador-cdu.js` (`resolverCDU`) normalizes everything to CDU while minimizing AI: **learned-cache → external-API stub → AI**. OpenLibrary's Dewey/LCC codes are captured; on an AI derivation the Dewey/LC→CDU mapping is **learned** into the `equivalencias_cdu` collection (`sistema_origen`, `codigo_origen`, `cdu`, `fuente`, `verificado`) so future books with that code are a free, consistent cache hit. Manual edits set `verificado: true`.

**Human-readable descriptions** of classification codes are cached + AI-generated on demand (`utils/descripcion-cdu.js` · `describirCDU` → `cdu_descripciones`; `utils/descripcion-clasificacion.js` · `describirClasificacion` → `clasificacion_descripciones` for Dewey/LCC). The panel ficha + Dashboard show each code with its **concise title + a ⓘ** (popup with the exhaustive text, `GET /api/clasificacion?sistema=&codigo=` — generates+caches on a miss) **+ a count** of docs sharing that code (click → Búsqueda filtered exactly, `/catalogo?clasSistema=&clasCodigo=`). Missing descriptions are **backfilled lazily during maintenance** (`mantenimiento/backfill-descripciones.js` · `rellenarDescripcionesFaltantes`, ~`MANTENIMIENTO_DESC_LOTE`=5 codes/pass; the Conformador stays awake until none remain), or in one shot with `node scripts/describir-clasificaciones.js --ejecutar` (CDU+Dewey+LCC; `describir-cdus.js` is the CDU-only predecessor).

## Outputs & file management

After insert, files are copied to `PATH_CDU/<cdu>/<libros|revistas>/<isbn|issn|ObjectId>/` alongside cover images, plus `registro.json` (resolved names) and `registro.marc.xml` (MARC 21 / MARCXML, via `src/marc21.js`). **CDU folder names are sanitized for Windows+Linux** (`utils/rutas.js`: `141.78:81'37` → `141.78_81`) while the real CDU string stays in Mongo. The doc's `ruta_base`/`imagenes`/`portada` point at `/recursos/...` (served statically by `app.js`).

## Search: local SQLite FTS index (`utils/indice-busqueda.js`)

The panel's Búsqueda (`GET /api/catalogo?q=…`) is accelerated by a local **SQLite FTS5** index (`busqueda.db`, same `better-sqlite3` already used by the Fichero — C, no SIMD, Atom-safe). **Mongo stays the source of truth**; the index only resolves the *text* match: it's fast and **accent/case-insensitive** (`tokenize='unicode61 remove_diacritics 2'` → "matematicas" finds "Matemáticas"). `/catalogo` is **hybrid**: with `q` and the index available it gets the bm25-ranked `_id`s and feeds them into the *existing* Mongo pipeline as an `_id $in` constraint (so **every structured filter — nsfw, valoración, CDU, dashboard — is untouched**) sorted by relevance; **if the index is missing/unavailable it falls back to the old Mongo `$regex`** (search never breaks). Identifiers (ISBN/ISSN, separator-tolerant + series-ISSN-via-colección) always go through Mongo, unioned with the FTS hits. The index self-maintains at the mutation points (ingest in `servicio-ingesta`, manual edit in `editar-doc`, delete in `reproceso·desvincularYReciclar`) via `indexarDoc`/`desindexarDoc` (best-effort, never throws); a full rebuild is `node scripts/reindexar-busqueda.js` or the panel's **Búsqueda** card (`POST /api/busqueda/reindexar`, background + progress like Integridad). The `.db` is **derived/rebuildable** — not in git, not in the deploy rsync; default path is next to `fichero.db` (`PATH_BUSQUEDA` overrides). Run a reindex once after the first deploy to populate it.

**Discover (search *outside* your library):** an opt-in **toggle** in the panel search (`🔭 Fichero`, admin-only) queries the **Fichero** dump (OL+BNE, 58.7 M) by full text via its existing `fichero_fts` FTS5 (title/subtitle/author — *not* subjects) → `GET /api/descubrir?q=`. **Critically it runs in a WORKER THREAD** (`utils/fichero-worker.js` ← `utils/fichero-descubrir.js` client) because `better-sqlite3` is **synchronous**: a heavy 58 M-row FTS on the main thread blocked the event loop and made the *library* search `/catalogo` time out (504). It's also **on-demand** (a button / Enter, never per-keystroke) so it never competes with the prioritized library search, and returns up to **100 results with client-side pagination**. Results render as a **distinct list below** the library results (no covers), each row badged `✓ la tienes` if its ISBN is already in `biblioteca`, with **acquisition links** built from `FUENTES_COPIA` (Anna's/Z-Lib/Libgen/Gutenberg — same sources as the ilegibles fix-it). It never ingests directly: you download → drop in the Inbox. Degrades to "no disponible" if the worker can't start or `fichero.db` lacks the FTS table. (`buscador-local.js·buscarTextoEnFichero` is the query itself, called *inside* the worker.)

## Panel web (PWA) — client UI & browser-side subsystems (`public/index.html`, one file)

The daily UI is a single-page panel served by `app.js` (static `public/`), **mobile-first** and an installable **PWA** (`manifest.json` + `sw.js`). Auth (`src/auth.js`) is **stateless HMAC tokens** (survive container restarts, ~2-day TTL); `autenticar` blocks every non-GET for non-admins (global mutation guard) and guests never trigger AI. Pages (en este orden en el menú: biblioteca → trabajo → excepciones → conjunto): Catálogo, Obras, Colecciones, Autores, Editoriales, Ubicaciones, **Entrada** (alta de recursos; id interno `inbox`), **Mantenimiento** (estado del sistema; id interno `activity`), Cuarentena, Papelera, Dashboard. Los ids internos NO se renombraron a propósito (deep-links, `loaders`, historial). La tarjeta de **Logs es ÚNICA** y `go()` la MUEVE al final de Entrada o Mantenimiento (duplicarla repetiría ids y rompería el visor); en el resto de páginas se para su temporizador.

- **Cost-minimizing CV runs in the BROWSER (no sharp/AI):** scanned-PDF→page-JPGs via **pdf.js**, **barcode** read via native `BarcodeDetector`, and the whole **Tapete** (green/teal/pink ruled cutting-mat) pipeline: `detectarBordesVerde` (mat mask → **dilate to swallow grid lines** → flood-fill exterior → **largest connected non-mat component = the book** → 4 diagonal-extreme corners, robust to grid leaks / inner mat-colored holes / outside clutter), `detectarRejillaPxCm` (grid pitch via **gradient-orientation histogram → fine angle search projecting perpendicular to the lines**, robust to a tilted mat; px/cm from the 1 cm cell), `rectificarLibro` (homography crop+deskew). **Mat color = `_verdeMat`**, by **HUE** when a calibration exists (`🎨 Calibrar` from an empty-mat photo, any colour/light, stored in localStorage) else a green/teal heuristic. `🔍 Probar` overlays the mask + detected quad + measure. Measurement is **rotation-invariant** (corner distances ÷ px/cm); dims saved via `POST /documentos/:id/dimensiones`. Switch `📐 Tapete` in the Inbox does measure+crop on upload, fire-and-forget; `🖼️ Elegir portada` lets you pick the cover before sending (barcode image = back).
- **Inbox flow:** Supervised (open ficha to accept, or **Cancel → `POST /documentos/:id/descartar`** which deletes a *recent* alta without password — recycles to Papelera) vs Autopilot (queue, non-blocking). Per-job process messages (ISBN/tapete/vision) persist **under each job** in the list.
- **NFC (Web NFC, Android/Chrome):** book tags carry the `?doc=` deep-link **plus an offline `BIB1:{json}` text record** (ubicación·título·isbn·cdu·colección·autor·editorial, fixed char caps so it never overflows NTAG215); shelf tags use `?amb=&est=` (lists the shelf). Reading **tries the ficha (timeout) then falls back to the offline card** (location prominent) — never relies on `navigator.onLine`.
- **Offline PWA:** `sw.js` is **network-first** for the app shell (online = fresh, deploys land; offline = cached shell) — `/api` and `/recursos` never cached; boot starts in **offline mode** (stored token+role) so NFC reading works with no server.
- **New client-touching server bits:** `src/utils/imagenes-doc.js` (carousel reorder/add/del/replace + edited-image replace), `src/utils/gestion-ubicaciones.js` (`ubicaciones` registry), `/ajustes/guest-nsfw` (guest NSFW toggle), `/documentos/eliminar-lote`, `/documentos/:id/{dimensiones,descartar,imagenes/*}`, `/catalogo?...&nsfw=1&soporte=&ambito=&estanteria=&estrellas=`. Legacy heavy files still apply; **edit `public/index.html` for any panel change** (it's the whole front-end).

## Failure handling

`src/errores.js` defines the taxonomy; `src/gestor-fallos.js` routes:
- **`ErrorIdentificacion`** (no title after all tiers) → **Cuarentena/no-identificados** (move out, manual review).
- **illegible / structurally-damaged file (corrupt PDF/EPUB) or 0-byte ghost** → **Cuarentena/ilegibles** (`enviarAIlegibles`, deposit + sidecar): not a cataloguing problem but a *needs-a-healthy-copy* one — fixed from the panel's Cuarentena fix-it workflow (`src/utils/saneamiento.js`): search-link a copy (Anna's Archive/Z-Library/Libgen/Gutenberg, configurable `FUENTES_COPIA`) → **upload it** (`POST /api/saneamiento/reemplazar`: validated by size + magic-bytes and **staged** in the deposit's `.reemplazo/`, marked `estado.listo` — *not catalogued yet*) → **select the ready ones + Procesar** (`POST /api/saneamiento/procesar`: a **background sequential** job catalogues each via the pipeline and recycles its deposit; progress at `GET /api/saneamiento/estado`). Ilegibles have **no Reingestar** button (a broken file can't be reprocessed into health) and the server refuses to reingest 0-byte files. Any deposit can also be **discarded** — its whole folder is moved **intact** to the Papelera under its category (easy to restore), per-item (`POST /cuarentena/descartar`) or per **whole category** (`POST /cuarentena/categoria/descartar`, which re-confirms the admin password). Category cards in the panel are foldable (persisted in `localStorage`) with a per-category "select all ready" checkbox.
- **`ErrorInfraestructura`** (can't reach a resource) → **Reintentos** (copy, retry later).
Each writes an `estado.json` with work-so-far for resume. Bibliographic APIs **degrade gracefully** (alert + continue) on a network blip; because MongoDB is **Atlas (cloud)**, a genuine connectivity outage surfaces as a Mongo error → Reintentos.

## The MongoDB `biblioteca` collection is `$jsonSchema`-validated

Required: `tipo_recurso` (`libro|revista`), `titulo`, `cdu`, `idioma`, `formatos` (enum array), `ubicacion` (`{ambito, estanteria}`). `autores` is an array of ObjectId, `editorial` an ObjectId. Identifiers are checksum-validated (`utils/identificadores.js`) and dropped to keep the record `pendiente` if malformed. A schema violation throws error 121 — never persist null/empty optional fields.
